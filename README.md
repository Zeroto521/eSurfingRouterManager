# 🎛️ 天翼路由器控制台

[![GreasyFork Version](https://img.shields.io/greasyfork/v/583456.svg)](https://greasyfork.org/zh-CN/scripts/583456)

> 🚀 **重塑旧时光**：专为 2016 年代老旧天翼光猫（电信网关）打造的现代化前端UI，接管并重构了原生极其简陋且 Bug 频出的光猫管理页面。

![预览图片](./assets/main.png)

## ✨ 核心特性

### 📡 局域网资产雷达

* **设备标识**：彻底告别 `anonymous`！支持为局域网设备自定义**名称、类型、品牌、系统**。
* **记忆设备**：即使设备生成全新随机 `MAC`，也能通过原生主机名指纹，一键下拉无缝继承旧版设备标识。
* **标识导出**：支持将标识数据导出为 `JSON`，并支持跨浏览器一键导入恢复。

### 📈 网速图表 UI 优化

* **渲染优化**：接管 `Chart.js` 老旧引擎，避免数据膨胀导致图表损坏。
* **展示改善**：过滤无用的 `00:00:00` 占位点；当历史数据点超过 40 个时，自动隐藏密集的网速节点，仅保留网速曲线。
* **单位适配**：实时拦截反人类的 `Mbps` 网速单位，自动转换为 `KB/s` 和 `MB/s`。

### 🛠️ Wi-Fi 控制与管理

* **数据修改**：支持自定义 `Wi-Fi` 名称，脱离 `ChinaNet-` 前缀命名。
* **频宽锁定**：一键锁定 `40MHz` 频宽（配合主辅信道捆绑逻辑）或安全降级为 `20MHz`。
* **协议净化**：强制锁定纯净的 `WPA2-PSK` 与 `AES` 加密模式。彻底斩断老光猫为向下兼容而保留的 `TKIP` 混合模式，不仅消除了“低安全性”弹窗警告，更释放了光猫吞吐潜力。

## 💻 脚本兼容

脚本针对 2016 年代中国电信大规模部署的老旧光猫进行了深度逆向与适配。

**目前已实机高强度测试通过的硬件环境：**

* **产品名称**：以太网无源光纤接入用户端设备 (EPON ONU)
* **设备类型**：EPON 上行天翼网关 (2+1)
* **设备型号**：**PT921** (2016年9月生产)
* **硬件版本**：`PT921V3.0`
* **软件版本**：`V1.0.921R003`

## 📦 安装与使用

### 一键安装

1. 确保浏览器已安装 [Tampermonkey](https://www.tampermonkey.net/) 扩展程序。
2. 访问 [Greasy Fork 脚本主页](https://greasyfork.org/zh-CN/scripts/583456)。
3. 点击“安装此脚本”，完成后登录天翼网关后台 ([192.168.1.1](http://192.168.1.1)) 即可体验。

### 手动安装

1. 安装浏览器扩展 [Tampermonkey](https://www.tampermonkey.net/)。
2. 在 Tampermonkey 中新建脚本，将本项目中最新的 [`src.user.js`](./src.user.js) 源码复制并保存。
3. 登录天翼网关后台 ([192.168.1.1](http://192.168.1.1))。
4. 控制台面板会自动悬浮在页面 **右下角** 即可开始使用。
